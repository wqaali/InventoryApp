Docker is an open-source platform for developing, shipping, and running applications using containers.
 It packages an application and all its dependencies (code, runtime, system tools, libraries, and settings) 
 into a single, standardized, isolated unit, ensuring it runs consistently across any environment, from a developer's laptop to the cloud.
 
 A dockerfile is a script file that contains a set of instructions used to create a docker image.
 
Kubernetes is an open-source container orchestration platform. It helps in automating the deployment, 
scaling, and management of containerized applications. For example, if I have multiple Docker containers running across servers,
Kubernetes ensures they are running properly, can scale them up or down based on load, and can automatically recover if something fails.

->Kubernetes Deployment
A deployment in kubernetes manages the creation,scaling, and updating of Pod,ensuring the application runs as specified.
It automates rolling updates,rollbacks,and scaling to maintain the desired state of the application.

-> Apply Deployment
Go to folder wher deployment file exist and run that command "kubectl apply -f deployment.yaml"
Decribe Deployment -> kubectl describe deployment inventory-deployment

-> Kubernetes Service
Service in Kubernetes provides a stable network endpoint for a set of pods, allowing consistent access through a fixed IP and DNS Name
It handles load balanceing and can be configured for internal or external access.
